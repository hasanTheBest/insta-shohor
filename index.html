Javascript Conceptual Topics

1. How does JavaScript Work?

JavaScript runs in browser environment powered by an engine. The V8 is the most popular JS engine which is developed by Google and used by Chrome and NodeJs.

Browser engine’s take the raw JS code and parse line by line in the direction top to bottom and produce an Abstract Syntax Tree (ABS). If the given code does not follow proper syntax it throws the syntax error at this stage.

The engine ignites interpreter to interpret Syntax Tree and produce ByteCode (machine code) which is appropriate to understand by the machine. Machine runs the piece of code and JavaScript performs it task.

In modern engine like V8 performs Just In Time (JIT) and optimize ByteCode to produce faster to run and efficient machine code.

2. What does JavaScript Event Loop Do?

JavaScript is a single threaded dynamically typing scripting language. It provides Synchronous, non-blocking and Asynchronous, blocking behavior. In synchronous mode JS engine takes a task and execute the task and the process is going on until the completion all the task.

Every browser consists of two parts the engine and a WebApi. The WebApi helps performing asynchronous task such as loading file from local disk, making a network request to load data.

JavaScript’s engine handle asynchronous task by assigning it to WebApi and continuing to next execution of Callstack. WebApi process the assigned task and finally push the task to Callback queue.

Event loop makes interaction between callback queue and callstack. After running all the synchronous task callstack becomes free then event loop run itself and put a task into callstack for execution until the callback queue is empty.

3. What are the defference between Local Storage and Session Storage?

Local storage and Session is similar to but differs in terms of saving data for next use. The Local storage read-only property of the window interface allows us to access a storage object for the document’s origin; the stored data is saved across browser sessions.

The read-only Session storage property accesses a session storage object for the current origin. Data in session storage is cleared when the page session ends.

Local storage data is specified to url protocols. Data in local storage is cleared while browsing in private window and closed the last tab. It may be undefined the when the file loaded from local server or computer because various browser handle it different way.

Opening a tab or window creates new session storage while closing clear the storage data. Multiple tab or window create multiple instances even the url is same. But clone of the tab does not create new Session storage.
